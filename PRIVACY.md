# 개인정보 및 데이터 수집 고지

이 문서는 HyperDesk가 사용자 PC에서 어떤 데이터를 읽고, 어떻게 사용하는지
설명합니다. 실제 상용 출시 전 이 내용을 정식 개인정보처리방침 양식으로
변환하고 법무 검토를 받으세요.

## 수집/조회하는 데이터

| 데이터 | 출처 | 용도 | 외부 전송 여부 |
|---|---|---|---|
| RDP 접속 기록 (호스트명) | `HKCU\Software\Microsoft\Terminal Server Client` | 대시보드에 원격 호스트 자동 등록 | 없음 (로컬 전용) |
| Horizon/Omnissa 서버 목록 | `HKCU\Software\[VMware\|Omnissa]\...` | 대시보드에 VDI 호스트 자동 등록 | 없음 (로컬 전용) |
| Hyper-V VM 목록/상태 | PowerShell (`Get-VM` 등) | VM 모니터링/제어 | 없음 (로컬 전용) |
| 시스템 리소스(CPU/메모리/디스크) | `sysinfo` crate | 대시보드 텔레메트리 | 없음 (로컬 전용) |
| 사용자가 직접 추가한 호스트 메모/태그 | 앱 내 입력 | `%APPDATA%/HyperDesk/hosts.json`에 저장 | 없음 (로컬 전용) |

## 원칙

- HyperDesk는 위 데이터를 외부 서버로 전송하지 않습니다. 모든 처리는
  사용자의 로컬 머신에서만 이루어집니다.
- RDP 비밀번호는 저장하지 않습니다. `mstsc.exe` 실행 시 사용자명만 임시
  `.rdp` 파일에 기록되며, 해당 파일은 연결 시작 10초 후 자동 삭제됩니다.
- 레지스트리 스캔(`get_dashboard`)은 사용자가 명시적으로 앱을 실행하고
  대시보드를 열 때만 수행되며, 백그라운드 상시 수집은 하지 않습니다.

## 엔터프라이즈 배포 시 주의

사내 IT 정책상 "엔드포인트의 레지스트리 접속 기록을 읽는 소프트웨어"에 대한
사전 고지/동의 절차가 필요한 조직이 있을 수 있습니다. B2B 납품 시 고객사
정보보안팀에 이 문서를 사전 공유하는 것을 권장합니다.
